# Auto Memory Sync Workflow

> **定位**：Layer 1 写入器。轻量、快速，只负责忠实记录"发生了什么"，不要求精确分类或深度抽象。
> 需要时再通过 `/memory-consolidation` 将碎片提纯为 `.memory/` 规则。

---

## 触发时机

任务完成后，若本次对话存在以下任一信号，则触发本工作流：

- 用户手动修改了 AI 生成的代码
- 用户口头否定了某个方案（"不要/改成/不这样"）
- AI 在执行中发现与记忆库冲突并进行了判断

---

## 步骤详解

1.  **收集纠错信号**
    - 回顾本次任务：用户的手动改动 + 口头否定反馈。
    - 通过 `view_file` 或 Git 差异，对比 AI 原始生成与最终版本之间的差异。
    - 初步判断每处修改的意图类型（偏好修正 / 规范落地 / 功能纠偏 / 业务补充 / 一次性）。

2.  **主动询问用户（有记录价值时必须执行）**

    > 「这次发现了 [一句话描述]，我想把它记录到对话日志中，后续整理时再提炼为规则。可以吗？」

3.  **写入 `.logs/session-YYYY-MM-DD.md`**（用户确认后）
    - 文件不存在则创建；已存在则追加。
    - 按以下格式写入：
      ```markdown
      ## [HH:MM] [任务简述]

      - **纠错/修改**: [用户改了什么，或明确否定了什么]
      - **用户意图**: [偏好修正 / 规范落地 / 功能纠偏 / 业务补充]
      - **原始上下文**: [可选，1 句话背景]
      ```
    - 一次性修改直接跳过，不写入。
