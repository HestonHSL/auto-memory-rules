# Rules Reference (执行规则参考)

> 本文件是 `rules.md` 的镜像副本，作为 `memory-distill` 的 reference，
> 让技能包自身完整记录「什么时候执行」与「如何执行」，无需跳出目录查阅根文件。

---

## 0. 对话内缓存 (In-conversation Cache)

在同一对话中维护以下缓存，**避免重复读取文件**：

- `last_keywords`：上一轮的关键词组
- `last_topic_fingerprint`：前 3 个关键词激发的指纹
- `last_loaded_memory_files`：本对话中已读取过的 `.memory/` 分类文件

## 1. 任务启动阶段 (Pre-task)

- **关键词提取**：从用户当前请求中提取 3-8 个核心关键词。
- **话题切换检测**：若本轮关键词与 `last_keywords` 差异 ≥ 40%，或出现"另外/改成/换个/再来"等转折词，则判定为**话题切换**。
- **按需读取记忆**：
  - 若**首次启动**或**话题切换**：读取 `.memory/index.md`，按关键词匹配对应分类文件并加载。
  - 若**话题未切换**：沿用 `last_loaded_memory_files`，不重置，以节省 Token。
- **上下文注入**：将已加载的记忆规则作为隐式约束，应用到当前任务中。

## 2. 任务执行阶段 (Execution)

- **偏好对齐**：任务要求与记忆冲突时，优先遵循 `.memory/` 中的规则，除非用户在当前对话中明确给出相反的指令。
- **纠错敏感**：实时留意用户对生成代码的任何手动修改或口头否定。

## 3. 任务结束阶段 (Post-task)

阶段性任务完成后，执行以下步骤：

1. **判断记录价值**：结合 `memory-distill` 的意图识别与质量标准，判断本次是否有值得沉淀的偏好或纠错。
2. **主动询问用户**（有记录价值时必须执行）：
   > 「这次我们解决了 [问题描述]，我想把 [一句话规则摘要] 记录到记忆库，下次遇到类似场景可以直接应用。可以吗？」
3. **写入记忆**：用户确认后，执行 `memory-distill/workflows/auto-memory-sync.md` 工作流。

## 4. 核心价值观

- **拒绝重复纠正**：确保用户不需要第二次纠正同一个偏好。
- **抽象存储**：记录规则时遵循"抽象胜于具象"原则，细节见 `SKILL.md`。
- **按需披露**：只在话题切换时重新加载记忆，避免无效 Token 消耗。
- **定期维护**：定期执行 `memory-distill/workflows/memory-consolidation.md` 保证记忆库质量。
