# 记忆系统：自动学习用户偏好

## 最高优先级

**用户的实时要求 > 历史记忆** — 冲突时无条件执行用户要求，对话结束时询问是否覆盖旧记忆。

## 触发信号

检测到以下任一信号时，立即进入记忆模式：

**纠错类**：注意、不对、错了、应该是、改成、换成、不要、还要、别忘了、漏了
**偏好类**：我更喜欢、最好、建议、我的习惯、我通常、以后要
**记忆类**：记住、记录、下次注意、形成规则、保存偏好

**隐式触发**（无明确关键词但意图明确）：
- 通用范围词：**统一**、**所有**、**所有的**、**都+动词**、**任何**、**每个**、**每次**、**全部**、**一律** — 用户在建立全局规则
- 同一问题被纠正 2 次以上 — 主动询问是否记录

隐式触发时主动提示：
```
💡 检测到可记录的记忆：[简短描述]
是否要记录到记忆系统？这样下次我会自动应用。[Y/n]
```

## 执行流程

### 1. 对话开始
- 自动加载 `{SKILLS_PATH}/auto-memory-rules/memory/index.md` 查看历史规则
- 根据任务关键词匹配相关记忆，主动规避已知错误

### 2. 对话中（核心）

**三层反思**（创建记忆前必须执行）：
1. 表层：用户说了什么？
2. 深层：为什么会这么说？我哪里做得不够好？
3. 原则：背后的通用规则是什么？能泛化到类似场景吗？

**立即行动**：
- 创建记忆文件到 `{SKILLS_PATH}/auto-memory-rules/memory/YYYY-MM-DD-{category}-{brief}.md`
- 使用 SKILL.md 中的标准模板和分类（api/types/component/hook/state/backend/database/infra/testing/security/pattern/convention/quality/workflow/general）
- 告知用户："我已记录这条记忆，下次会自动应用"

### 3. 对话结束

如果创建了新记忆，运行（已安装 hook 则自动触发，无需手动执行）：
```bash
bash {SKILLS_PATH}/auto-memory-rules/scripts/generate-memory-artifacts.sh
```

## 核心原则

1. **每次触发都反思** — 理解深层原因，不只记录表面要求
2. **只记录可复用的** — 一次性特殊处理不记录
3. **冲突时询问** — 规则演进则覆盖，场景分支则合并并分别描述触发条件

## 记忆质量标准

应该记录：
- ✅ 用户明确的偏好表达
- ✅ 重复出现的修改模式（2次+）
- ✅ 全局范围的规范（"对于所有..."）
- ✅ 可抽象为通用原则，跨项目可用

不应记录：
- ❌ 一次性的特殊处理（"临时..."、"这里特殊..."）
- ❌ 项目特定的业务逻辑
- ❌ 临时的解决方案

详细实现参考 `{SKILLS_PATH}/auto-memory-rules/SKILL.md`
