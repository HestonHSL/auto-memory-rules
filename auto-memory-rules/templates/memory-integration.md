# 记忆系统：自动学习用户偏好

## 触发信号（立即启动记忆系统）

检测到以下任一信号时，立即进入记忆模式：

**纠错类**：不对、错了、应该是、改成、换成、不要、还要、别忘了、漏了
**偏好类**：我更喜欢、最好、建议、我的习惯、我通常、以后要
**记忆类**：记住、记录、下次注意、形成规则、保存偏好

## 执行流程

### 1. 对话开始
- 自动加载 `{SKILLS_PATH}/auto-memory-rules/memory/index.md` 查看历史规则
- 根据任务关键词匹配相关记忆，主动规避已知错误

### 2. 对话中（核心）
当用户提出修改/纠错时：

**立即反思**：
- 为什么用户会这么说？我哪里做得不够好？
- 背后的通用原则是什么？（质量/偏好/实践/规范）
- 这个原则可以泛化到其他场景吗？

**立即行动**：
- 创建记忆文件到 `{SKILLS_PATH}/auto-memory-rules/memory/YYYY-MM-DD-{category}-{brief}.md`
- 使用 SKILL.md 中的标准模板和分类（api/hook/component/types/state/pattern/convention/quality/workflow）
- 记录反思过程和提取的原则

### 3. 对话结束
如果创建了新记忆，运行：
```bash
bash {SKILLS_PATH}/auto-memory-rules/scripts/generate-memory-artifacts.sh
```

## 核心原则

1. **用户当前请求 > 历史记忆** - 冲突时优先执行用户请求，询问是否更新记忆
2. **每次纠错都反思** - 理解深层原因，不只记录表面要求
3. **只记录可复用的** - 一次性特殊处理不记录
4. **持续学习改进** - 目标是让相同错误越来越少

## 记忆质量标准

应该记录：
- ✅ 用户明确的偏好表达
- ✅ 重复出现的修改模式（2次+）
- ✅ 可抽象为通用原则
- ✅ 跨项目可用的最佳实践

不应记录：
- ❌ 一次性的特殊处理
- ❌ 项目特定的业务逻辑
- ❌ 临时的解决方案

详细实现参考 `{SKILLS_PATH}/auto-memory-rules/SKILL.md`
