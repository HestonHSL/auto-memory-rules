# Memory-Distill

让 AI 记住你的每一次纠错，永远不需要第二次纠正同一个问题。

Memory-Distill 是一套面向 AI Native 开发的长期记忆管理系统。它通过一套精心设计的「蒸馏」机制，将你在开发过程中对 AI 的零散纠错转化为可持久执行的项目规则。

## 🎯 解决的痛点

- **重复劳动**：每次新任务都要重新告诉 AI 你的命名偏好或库选型。
- **瞬时记忆**：AI 在长对话末尾容易忘记开头的约束。
- **环境迁移**：换一个 IDE 或项目，AI 就不再「懂你」。

## 🏛️ 核心架构：两层记忆模型

系统将记忆明确分为两层，实现快速捕捉与高质量沉淀的平衡：

1. **Layer 1：对话碎片 (.logs/)**
   - **特点**：原始、具象、快速记录。
   - **用途**：在任务结束时，忠实记录用户改了什么，作为后续提纯的原材料。
2. **Layer 2：提纯规则 (.memory/)**
   - **特点**：抽象、指令化、全局生效。
   - **用途**：由 AI 过滤掉一次性修改，提炼出跨任务通用的「硬规则」。

## 🛠️ 工作流 (Workflows)

系统内置两个核心自动化工作流：

- **`auto-memory-sync`**：
  在每个任务完成后执行。AI 会自动对比差异并询问用户，将有价值的纠错记录到 `.logs/`。
- **`memory-consolidation`**：
  定期执行。AI 将 `.logs/` 中的零散碎片按[意图识别协议](./memory-distill/references/distillation-logic.md)进行蒸馏，更新 `.memory/` 中的权威规则库。

## 🔍 什么是语义化搜索 (Semantic Search)？

在本项目中，我们讨论了语义化搜索（如使用 QMD 工具）作为未来扩展的方向：

- **普通匹配 (Keyword Match)**：搜索「按钮样式」，只能找到标题里有「按钮」的文件。
- **语义搜索 (Semantic Search)**：通过向量（Embedding）技术，搜索「如何处理加载反馈」能自动匹配到「在提交时应显示 Spinner 动画」的条目——即便两者没有任何共同关键词。
- **本项目策略**：当前版本优先使用**基于关键词的渐进式披露**。当你的记忆库条目超过 50 条，关键词开始失效时，建议引入 QMD 等工具进行语义化升级。

## 📁 目录结构

```text
.
├── rules.md                   # 核心行为准则（须放在项目根目录）
└── memory-distill/            # 技能包核心
    ├── SKILL.md               # 导航中心
    ├── workflows/             # 自动化脚本（同步与整理）
    └── references/            # 逻辑法典（意图表、质量门控）
```

## 🚀 如何安装

1. 将 `memory-distill/` 文件夹拷贝到你的项目根目录。
2. 将 `rules.md` 拷贝到你的项目根目录（或将其内容合并到你的 `.cursorrules` / `rules.md` / `.clauderules` 中）。
3. 开启 AI 编程助手，它将自动根据 `rules.md` 开始运作。
