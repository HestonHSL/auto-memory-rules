# QMD 语义化检索升级指南 (Future Upgrade)

> 当你的 `.memory/` 规则库超过 50 条，关键词匹配开始出现遗漏或检索效率低下时，建议按照本指南引入 QMD 语义搜索。

---

## 1. 什么是 QMD？

**QMD (Query Markdown)** 是一个命令行工具，它能将你的 Markdown 文件向量化（Embedding），存入本地向量数据库，并提供语义检索功能。

- **关键词匹配**：只能搜索文件中出现的「具体文字」。
- **语义检索**：可以搜索文件的「意思」。例如输入"视觉反馈规范"，即便文档里只写了"交互提示"，QMD 也能通过语义相关性找到它。

---

## 2. 升级后的架构变化

引入 QMD 后，记忆注入的逻辑将从「读取目录/索引」变为「向量查询」：

| 模块           | 当前方式 (Keyword-based) | 升级后 (QMD-based)                   |
| :------------- | :----------------------- | :----------------------------------- |
| **规则存储**   | `.memory/*.md` 平铺      | `.memory/*.md` + QMD 向量索引        |
| **检索逻辑**   | 按分类文件名匹配关键词   | 使用 `qmd_query` 进行语义召回        |
| **上下文注入** | 注入整个对应分类文件     | 仅注入 QMD 召回的最相关的 3-5 个条目 |

---

## 3. 如何实施升级

### 第一步：安装 QMD

在本地环境中安装 QMD CLI：

```bash
npm install -g qmd
```

### 第二步：建立索引

对你的 `.memory/` 目录进行向量化：

```bash
qmd collection add project-memories --path .memory/ --name "My Project Memories"
qmd embed project-memories
```

### 第三步：修改 rules.md 指令

将 `rules.md` 中的「按需读取记忆」章节修改为：

- **Pre-task (启动)**：
  - 执行指令：`qmd_query "当前任务相关的关键词或描述"`。
  - 获取 QMD 返回的最匹配 Markdown 片段。
  - 将这些片段作为 Prompt 注入，而非加载整个 `.md` 文件。

---

## 4. QMD 的优势

1. **解决 Token 焦虑**：不再加载整个文件，只加载最精准的几行记忆。
2. **模糊匹配能力**：即使用户的提问、AI 的理解和记忆的表述完全不同，只要意思接近，就能生效。
3. **支持跨语种**：语义引擎通常支持中英文互搜。

---

> **注意**：在记忆库较小时（<50条），关键词匹配速度更快且更直接，无需额外依赖。建议在感觉到"AI 开始不听话"或"Token 消耗过大"时再进行此升级。
